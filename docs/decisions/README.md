# Architecture Decision Records

This directory contains ADRs (Architecture Decision Records) for the Splash project, using the [MADR format](https://adr.github.io/madr/#example).

## When to Write an ADR

Any large change that involves technical trade-offs and warrants documentation should have a record here.

## Naming

Files should follow the pattern: `YYYY-MM-DD-title.md`

Example: `2024-03-15-effect-checking-strategy.md`

## Template

A starter template is available at [`.adr-template.md`](.adr-template.md).
